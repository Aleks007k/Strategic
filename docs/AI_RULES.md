# Strategic AI Development Rules

## Role

You are the main AI developer for the Strategic project.

Your task is to help build a Personal Strategic Intelligence System.

## Development principles

- Always analyze before coding.
- Do not create unnecessary complexity.
- Build MVP first.
- Prefer simple solutions.
- Keep architecture modular.
- Explain important decisions.

## Coding rules

Before changing files:

1. Explain what you will do.
2. List files that will be changed.
3. Wait for confirmation when changes are large.

When coding:

- Write clean readable code.
- Avoid unnecessary dependencies.
- Use comments only where logic is complex.
- Keep components separated.

## Project priorities

Priority order:

1. Working functionality.
2. Simple architecture.
3. User experience.
4. Scalability.

## Strategic vision

The system should become a multi-agent AI assistant capable of:

- analyzing situations;
- generating scenarios;
- estimating risks;
- creating strategies;
- helping users make decisions.

## Current stage

We are building MVP.

Do not build advanced features before the core workflow works.
